# XHS Local Helper Plan

## 目标

构建一个运行在用户本机的轻量 Helper，托管 GitHub 版 `xiaohongshu-mcp` 二进制，并通过 localhost HTTP API 给网页前端调用，完成登录、状态检测和发布。

## 范围边界

- 本地保存登录态，不经过 musegate 服务端
- 前端直接调用本地 Helper，不依赖浏览器插件
- 第一阶段仅支持 macOS Apple Silicon
- 底层发布能力复用 `xiaohongshu-mcp-darwin-arm64.tar.gz`

## 阶段 1：PoC 跑通

目标：在开发机本地完成最小闭环。

- [ ] 1.1 安装 Go 并完成项目首次编译
- [ ] 1.2 启动 Helper 服务并验证 `GET /status`
- [ ] 1.3 完成 `POST /install`，把 `xiaohongshu-mcp` 二进制解压到本地运行目录
- [ ] 1.4 完成 `POST /login`，能够拉起 `xiaohongshu-login-darwin-arm64`
- [ ] 1.5 完成 `POST /publish`，通过本地 MCP 成功发出测试帖子
- [ ] 1.6 固化日志、PID、cookies 路径约定

## 阶段 2：前端联调可用

目标：让网页前端可以稳定调用本地 Helper。

- [ ] 2.1 明确前端调用协议：`/status` `/install` `/login` `/publish`
- [ ] 2.2 加入 CORS 白名单和来源校验
- [ ] 2.3 增加短时 token 或签名机制，避免任意网页调用本地 Helper
- [ ] 2.4 约定前端状态机：未安装、未启动、未登录、发布中、发布成功、发布失败
- [ ] 2.5 提供联调示例请求和错误码清单

## 阶段 3：本地进程管理完善

目标：让 Helper 能长期稳定运行，而不是只适合开发调试。

- [ ] 3.1 增加 MCP 进程探活和自动拉起
- [ ] 3.2 增加登录进程状态检测和超时处理
- [ ] 3.3 增加日志滚动和错误诊断接口
- [ ] 3.4 增加 `/diagnose` 接口，返回安装、进程、日志、版本信息
- [ ] 3.5 处理 MCP session 失效后的自动重试

## 阶段 4：安装与交付

目标：让普通用户无需安装 Go，即可使用 Helper。

- [ ] 4.1 产出 `darwin-arm64` 可执行文件
- [ ] 4.2 设计安装目录与升级策略
- [ ] 4.3 封装成 `.app` / `.pkg` / `.dmg` 之一
- [ ] 4.4 处理 macOS 签名、公证、首次启动提示
- [ ] 4.5 增加开机自启或登录后常驻能力

## 阶段 5：产品化增强

目标：降低用户感知，提升稳定性和可维护性。

- [ ] 5.1 增加 Helper 版本自检和升级提醒
- [ ] 5.2 支持自动下载 `xiaohongshu-mcp` 指定版本
- [ ] 5.3 支持多架构识别
- [ ] 5.4 补充前端安装引导页和失败诊断文案
- [ ] 5.5 评估是否需要浏览器插件作为增强能力，而不是首发依赖

## 当前决策

- 本地 Helper 使用 Go
- 前端先直连 localhost Helper，不引入浏览器插件
- Helper 托管 `xiaohongshu-mcp`，不重写小红书发布协议
- 服务端不保存 XHS 登录态
